# 🚀 零基础上手：自动资讯监控与 PDF 简报生成系统

欢迎使用！这是一个**完全自动化**的机器人小助手。
它的工作是：每天早上按时起床，去您关心的网站（比如各种政府官网、新闻网站）转一圈，把今天刚刚发布的最新文章抓下来，重新排版成一本漂亮的 PDF 电子书，然后悄悄塞进您的邮箱里。

这份说明书专门为您打造，哪怕您是第一次接触编程，只要跟着下面 **1-2-3-4** 的步骤一步步来，也能在 5 分钟内让它为您工作！

---

## 🛠️ 第一阶段：准备工作（只需做一次）

要让这个机器人跑起来，您的电脑需要安装两个“翻译官”（Python 软件）。

### 1. 安装基础环境
请确保您的电脑上已经安装了 **Python**（建议版本 3.8 以上）。
打开您的“终端”（如果是 Windows 系统，可以在下方搜索栏搜 `cmd` 或 `powershell` 并打开）。

### 2. 安装机器人的“零件”
在终端里，把路径切换到咱们这个文件夹，然后复制下面这行代码并回车运行：
```bash
pip install -r requirements.txt
```
*(这会给机器人安装一些必需的零件，比如处理 PDF 的工具、发送请求的工具等。)*

### 3. 安装模拟浏览器
机器人需要一个隐形的浏览器去访问网页。继续在终端输入并回车：
```bash
playwright install chromium
```
*(这可能会花一两分钟下载一个浏览器，请耐心等待直到它提示成功。)*

---

## 🔑 第二阶段：配置属于您的密码本

机器人发邮件需要用您的邮箱，使用 AI 需要用您的密钥。为了安全，我们不能把密码直接写在代码里，而是要放在一个保险箱文件里。

1. 在当前文件夹下，**新建一个文本文档**。
2. 将它重命名为 **`.env`**（注意前面有个点，没有后缀名）。
3. 用记事本打开它，把下面这段文字复制进去，并把汉字部分**替换成您自己的真实信息**：

```ini
# ===== 邮箱自动发件配置 =====
SMTP_SERVER=smtp.qq.com
SMTP_PORT=465
SENDER_EMAIL=您的发件邮箱@qq.com
SENDER_PASSWORD=您的邮箱授权码（不是登录密码哦！）
RECEIVER_EMAIL=您的收件邮箱@qq.com

# ===== 大模型 API 配置 (用来让 AI 帮我们干活) =====
LLM_API_KEY=sk-您的真实密钥
LLM_BASE_URL=https://api.deepseek.com
LLM_MODEL_NAME=deepseek-chat
```
*(小贴士：QQ邮箱的“授权码”需要登录 QQ 邮箱网页版，在 设置->账户 里开启 SMTP 服务后获取。)*

---

## 🏃 第三阶段：在本地测试运行

环境都弄好了，我们来试运行一次看看效果吧！

1. 打开这个文件夹里的 **`main.py`** 文件，往下拉一点点，找到第 7 行：
   `TEST_MODE = False` 
   为了测试，请先把它暂时改成 **`TEST_MODE = True`**。
   *(测试模式下，它只会把做好的 PDF 保存在本地文件夹，不会真往邮箱里发邮件，方便我们调试。)*
   
2. 在终端里输入并回车：
   ```bash
   python main.py
   ```

3. 盯着屏幕，程序会自动去 `config.py` 里登记的网站抓取当天的文章。
   如果没有新文章，它会告诉您“今天没有新文章”。
   如果有，它会把文章抓下来，剔除广告，**如果有原生的 PDF 附件它也会自动下载**，最后把它们全部装订成一本名叫 `2026-x-x_每日监控简报合集.pdf` 的电子书！
   
4. **测试成功后**，千万别忘了把 `TEST_MODE` 改回 **`False`**！

---

## 🤖 第四阶段：如何添加新网站？（让 AI 帮您搞定）

系统目前在 `config.py` 里内置了几个政府网站的监控规则。如果您想监控全新的网站，以前需要您懂复杂的代码，现在**全权交给 AI 处理**即可！

**步骤如下：**
1. 找到 `urls.txt` 文件，打开它。把您想监控的新网页的链接粘贴进去（一行一个）。
2. 在终端里运行：
   ```bash
   python batch_add_sites.py
   ```
3. 这个名叫 `batch_add_sites.py` 的脚本会自动接入大模型，不到 10 秒钟，它就会把破解好的完美规则吐出来，保存在 `new_configs.txt` 文件里。
4. 打开 `new_configs.txt`，把里面生成好的 `{ ... }` 格式的代码复制。
5. 打开 `config.py` 文件，把刚刚复制的代码粘贴进 `SITES_CONFIG = [` 的中括号里面。大功告成！

---

## ☁️ 终极阶段：部署到云端，彻底解放双手！

如果您不想每天早起开电脑运行它，我们可以把它传到 **GitHub Actions** 上，让 GitHub 的超级计算机每天替我们打工！

1. 确保 `main.py` 里的 `TEST_MODE = False`。
2. 确保刚才产生的诸如 `__pycache__` 或 `output` 等无用的临时文件夹已经被删掉了（我之前已经帮您删过了，保持干净即可）。
3. 把这个文件夹里的所有文件，**上传到您的 GitHub 仓库**中。
   *(提示：您可以看到里面有一个名为 `.github/workflows/daily_scraper.yml` 的隐藏文件，这就是给 GitHub 云电脑看的作息时间表。)*
4. 上传完毕后，打开您 GitHub 仓库的网页。
5. 点击顶部的 **`Settings`** -> 左侧边栏找 **`Secrets and variables`** -> 点击 **`Actions`**。
6. 点击绿色的 **`New repository secret`** 按钮。
7. 您需要把刚才写在 `.env` 文件里的 5 个邮箱密码配置，一个一个手动建在网页上：
   - Name 填 `SMTP_SERVER`，Value 填 `smtp.qq.com`
   - Name 填 `SMTP_PORT`，Value 填 `465`
   - Name 填 `SENDER_EMAIL`，Value 填 您的发件邮箱
   - Name 填 `EMAIL_PASSWORD`，Value 填 您的授权码
   - Name 填 `RECEIVER_EMAIL`，Value 填 您的收件邮箱

配置完这五步后，一切就绪！GitHub 的云电脑会在每天早上 8:00 (北京时间) 自动启动，准时把一整天的情报汇总 PDF 发送到您的手机邮箱里。

祝您使用愉快！🎉
